# 启动说明

## 0. 配置 API 密钥（首次运行前）

各服务通过目录下的 `.env` 文件加载密钥。首次使用请确认以下文件已存在：

```bash
# AI 服务
ai-service/.env
DEEPSEEK_KEY=sk-xxx
TAVILY_API_KEY=tvly-xxx
SILICONFLOW_API_KEY=sk-xxx

# 用户服务
user-service/.env
STMP_EMAIL=your@163.com
STMP_EMAIL_PASSWORD=xxx
JWT_KEY=abcdefghijklmn

# 论坛服务
forum-service/.env
WEATHER_KEY=xxx
```

> 密钥可从 `my-project-backend/.env.properties` 迁移，或参考根目录 `.env.example`。
> `.env` 文件已配置在 `.gitignore` 中，不会提交到 Git。

## 1. 启动基础设施

```bash
cd docker && bash setup.sh
```

启动后确认 Nacos 控制台可访问：`http://localhost:8848`

## 2. 启动微服务（各开一个终端）

```bash
# 网关
cd gateway-service && mvn spring-boot:run    # 端口 8081

# 用户 + 存储
cd user-service && mvn spring-boot:run       # 端口 8082
cd oss-service  && mvn spring-boot:run       # 端口 8084

# 业务服务
cd forum-service && mvn spring-boot:run      # 端口 8088
cd notification-service && mvn spring-boot:run  # 端口 8085
cd announcement-service && mvn spring-boot:run  # 端口 8086

# AI 服务
cd ai-service && mvn spring-boot:run         # 端口 8083
```

API 统一入口: `http://localhost:8081` (网关)

## 3. 前端

```bash
cd my-project-frontend && npm run dev
```

访问: `http://localhost:5173`（前端 dev server）
