## 二、AI 应用情况报告

### 2.1 概述

本系统的开发过程中，团队**大规模使用 AI 工具**辅助完成从架构设计到编码测试的全流程工作。

### 2.2 使用的 AI 工具

| 工具 | 用途 |
|------|------|
| **Claude Code**（接入 DeepSeek 模型） | 代码生成、重构、调试、文档编写 |
| **OpenAI Codex / Claude Code** | 计划、算法实现、测试编写 |
| **Superpower Skills、Context7 MCP** | 项目规划、架构设计、代码审查、文档模板 |

### 2.3 AI 辅助开发场景

#### 2.3.1 架构设计与规划

- 使用 Claude Code 的 `/plan` 命令进行微服务拆分方案设计
- AI 辅助进行技术选型分析（Spring Cloud Alibaba vs Netflix、Vue 3 vs React 等）
- 使用 Superpower 的 `brainstorming` skill 进行需求梳理和功能规划

#### 2.3.2 后端代码生成

- **AI Agent 驱动的文档生成**：使用 Claude Code 的多 Agent 协作模式，并行生成系统需求文档（可行性分析、功能分析、领域模型）和设计文档（技术选型、系统架构、技术关键点），大幅提高文档产出效率
- **微服务骨架代码**：通过 AI 快速生成 Spring Boot 微服务的基础结构（Controller、Service、Mapper）
- **JWT 认证实现**：AI 辅助编写网关 JWT 过滤器、Redis 黑名单机制
- **Elasticsearch 搜索**：AI 生成 ES 查询 DSL、高亮显示配置
- **RabbitMQ 异步消息**：AI 辅助编写邮件队列生产/消费代码

#### 2.3.3 前端开发

- **Vue 3 组件开发**：AI 辅助生成 Element Plus 和 Vant 4 组件代码
- **SSE 流式渲染**：AI 实现前端 ReadableStream 逐行解析、打字机效果
- **移动端适配**：AI 辅助实现响应式布局、Vant TabBar/NavBar 组件集成
- **Quill 富文本编辑**：AI 配置 Delta JSON 序列化/反序列化

#### 2.3.4 AI 服务自身集成

- 使用 AI 工具测试和调试 DeepSeek API 的集成
- AI 辅助编写 Spring AI ChatClient 配置和 Tool 调用逻辑
- AI 生成 SSE 心跳机制和超时处理

#### 2.3.5 调试与排错

- AI 分析日志输出，定位微服务间 Feign 调用失败原因
- AI 辅助修复 ES 客户端版本兼容性问题
- AI 辅助解决跨域 CORS 配置问题

#### 2.3.6 文档撰写

AI 根据 Git 提交记录自动整理开发时间线，根据团队信息生成分工表，根据 AI 使用情况生成报告。
